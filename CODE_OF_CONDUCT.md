# Code of Conduct

## Our Pledge

Project participants are expected to keep this community respectful, practical, and welcoming. Discussion should focus on the work, the code, the gameplay, and the research ideas behind the project.

## Expected Behavior

- Be respectful when giving or receiving feedback.
- Assume good intent, but clarify when something is ambiguous.
- Critique ideas and code, not people.
- Keep discussions relevant to the project.
- Credit research, assets, and prior work accurately.
- Make room for contributors with different experience levels.

## Unacceptable Behavior

Unacceptable behavior includes:

- Harassment, intimidation, or personal attacks.
- Discriminatory language or conduct.
- Sexualized comments or imagery in project spaces.
- Trolling, insults, or sustained disruption.
- Publishing private information without permission.
- Misrepresenting research, authorship, or contribution ownership.
- Any conduct that maintainers reasonably determine to be inappropriate for the project.

## Scope

This Code of Conduct applies in project spaces, including issues, pull requests, discussions, commits, documentation, and any official communication channels connected to the project.

## Enforcement

Maintainers may remove comments, close threads, reject contributions, or restrict participation when behavior violates this Code of Conduct.

Enforcement should be proportionate to the situation and may include:

- A private or public clarification.
- A warning.
- Temporary limits on participation.
- Permanent removal from project spaces.

## Reporting

Report Code of Conduct concerns to the project maintainers through the repository's private contact method if available. If no private channel is available, open a brief public issue requesting maintainer contact without including sensitive details.

Maintainers should handle reports with care, avoid unnecessary disclosure, and act in the best interest of the project community.
